CSV Import/Export Guide

📋 How It Works

📌 First Step: Click "Export CSV" to download a template file with the correct format.

1

Export Template

Click Export CSV to download a file with the correct column headers. Open this file in Excel, Google Sheets, or any spreadsheet application.

2

Add Your Trades

Enter your trade data in the spreadsheet. The columns must remain in this exact order:

Type,Date,Symbol,Direction,Size,Entry,Exit,Stop Loss,Take Profit,P/L,Notes

Example:

Closed,2024-01-15,BTCUSDT,long,1,42000,43500,41500,44000,1500,"Great trade"
Active,2024-01-16,ETHUSDT,long,2,2200,,2150,2300,,"Still open"
3

Import Back

After editing, click Import CSV and select your file. Choose whether to replace existing data or append to it.

💡 Important Tips

  • Always export first to get the correct column format
  • For active trades, leave Exit and P/L columns empty
  • For closed trades, fill in both Exit and P/L
  • Use Replace mode to overwrite all existing data
  • Use Append mode to add new trades to your existing data
  • Keep a backup of your CSV files for safety

Step 2: Copy This Prompt 

Extract all trades from this trading platform screenshot.

FIRST: Before generating the CSV, confirm what you see. List the columns you found and show me the first 2 trades so I can verify.

ONCE I CONFIRM, generate the CSV in this EXACT format (copy this column order exactly):

Type,Date,Symbol,Direction,Size,Entry,Exit,Stop Loss,Take Profit,P/L,Notes,Breakeven

FORMAT RULES:
- Type: Always "Closed"
- Date: YYYY-MM-DD (if time not visible, use 12:00:00)
- Symbol: Use exact symbol (US30, ETHUSD, NAS100, etc.)
- Direction: "long" for Buy, "short" for Sell
- Size: Number (e.g., 0.12, 1.5, 2.0)
- Put comma between date and time (YYYY-MM-DD, 12:00:00)
- Entry/Exit: Number with 2 decimals
- Stop Loss/Take Profit: Number if exists, otherwise leave BLANK (nothing)
- P/L: Number with 2 decimals, negative if loss
- Notes: Leave BLANK (empty) for normal trades, put "REVIEW" if any number looks suspicious
- Breakeven: "No" normally, "Yes" if P/L is 0
- Help counting the trades & Calculate PnL
- Count how many trades to make sure its accurate with the picture                   
EXAMPLE:
Closed,2026-03-29,12:00:00,US30,short,0.08,45929.8,45709.6,45709.6,45468.6,17.62,,No
Closed,2026-03-29,12:00:00,ETHUSD,long,0.51,2060.20,2000.00,2000.00,2056.39,30.70,,No
Closed,2026-03-29,12:00:00,NAS100,long,0.14,24891.3,25001.13,24924.96,25001.13,15.38,,No

BLANK FIELDS RULE:
- If a field has no data (like no Stop Loss), leave it COMPLETELY EMPTY (nothing between commas)
- Example with blank SL: Closed,2026-03-29,ETHUSD,short,0.34,2156.94,2173.09,,,5.49,,No

- Just paste the CSV and tell me what to fix
- Once perfect, I'll download and import to TradeVault
